# user-progress

Spring Boot service for authentication and watch progress.

## Run (from `services/` parent)

    ./mvnw -pl user-progress spring-boot:run

Listens on http://localhost:8081

- Health: http://localhost:8081/user-progress/health
- Swagger UI: http://localhost:8081/swagger-ui.html
- Actuator health: http://localhost:8081/actuator/health

## Seed data

`V2__seed_demo_user.sql` seeds a demo account (applied automatically by Flyway):

| email | password |
|-------|----------|
| `demo@example.com` | `password123` |

The password is stored as a BCrypt hash generated by the app's own encoder, so it
works against `POST /user-progress/auth/login`. Demo use only — rotate or remove
before any production deployment. If you change the password, regenerate the hash
with the same `BCryptPasswordEncoder` (the seed won't work with a hand-edited hash).

## Test (from `services/` parent)

    ./mvnw -pl user-progress verify
